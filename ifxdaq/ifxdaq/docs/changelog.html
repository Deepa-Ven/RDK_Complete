Changelog

Changes for the upcoming release can be found in the Pull requests section in our repository.

4.0.0

2023-09-19 | v3.3.0..v4.0.0 | v4.0.0..develop

New features & improvements

  • Support for YOLOv8 - https://github.com/ultralytics/ultralytics. (#310, #314, #334, #335)

  • Add support for Python 3.10 & 3.11. (#321)

  • Support for RDK3.5. (#327)

  • Individual device classes for different radar sensors (RadarIfxBGT60TR13C, RadarIfxBGT60UTR13D, RadarIfxBGT60ATR24C). (#328)

  • Conditional imports for type checking to improve import performance and make type checking more robust. (#332)

Bugfixes

  • Reduce default frame rate for Intel RealSense cameras from 30 to 15 fps to support also less performant hardware. (#316)

  • Remove internal buffering to avoid data duplicates in radar data and improve performance. (#337)

Breaking changes

  • Drop support for Python 3.7. (#321)

  • RadarIfx* data shapes contain now an additional axis for TX antennas: [rx, chirp, samples] -> [tx, rx, chirp, samples] (#327)

Infrastructure

  • Add ruff for static analysis; replaces also flake8, isort & pydocstyle. (#311, #317)

1.0.0

2021-02-05

New features & improvements

  • Improved documentation. (#40)

  • Added Jenkins pipeline as CI. (#40)

  • ifxdaq supports now win_x64, win_x86, linux_x64, raspi_armv7l. (#40)

  • Fixed minor bugs and performance bottlenecks. (#40)

Backward-incompatible Changes

  • Refactored recorder package for future scalability. (#37)

    • Read & write functionality now unified under ifxdaq.fileio.

    • Separate file types for different use cases:

      • ìfxdaq.fileio.mp4: .mp4-files for RGB & BW videos.

      • ìfxdaq.fileio.npy: .npy-files for data streams, e.g. radar or depth.

      • timestamps in .csv-files.

    • HDF5 file format is now deprecated (still available under ifxdaq.fileio.core_legacy).

    • timestamps now in seconds for compatibility with other applications (previous: ms).

    • introduced generic abstraction layer in the complete package.

    • sensor offer a description of the frame format (dtype, shape, fps).

  • Updated default configuration files. (#40)

0.2.0

2020-12-21

New features & improvements

  • Added pylint for code analysis and enforcing of coding standards. (#15)

  • Added towncrier for automatic changelog creation. We test this for a few releases and evaluate afterwards if it is valuable. (#19)

  • Added support for RasPi. (#24)

  • Integrated Intel® RealSense™. (#31)

    • from ifxdaq.sensor.camera_irs import CamIntelRealSense.

    • Based on pyrealsense2.

    • Support RGB, depth and IR data streams.

    • Leverage configuration file format from RealSenseViewer.

  • Added a tool to record data with various sensors.

    • Invoke the application with ifxdaq recorder. (#34)

    • ifxdaq.sensor.sensor_manager.SensorManager to handle multiple sensors easily

      • Each sensor is spawned in a stand-alone process.

      • Configuration via OmegaConf and sensor-dependent configuration files.

      • Centralized logging with Python’s builtin logging module.

    • Stores recordings in a separate folder with the sensor config.

    • Not yet implemented: Meta-data information.

  • Added mypy for static type checking. (#35)

Backward-incompatible Changes

  • Renamed sensor classes to RadarIfxBGT60, CamOpenCV and CamCommonVisionBlox. (#22)

  • Separated playback functionality into own class.

  • Substituted classes in sensor.multiprocessing with SensorBackground wrapper.

  • Updated RadarIfxBGT60 to support latest configuration file format. (#27)

  • Refactored complete package architecture for future scalability. (#33)

    • ifxdaq.sensor: Sensor modules

      • Sensor implementations

        • camera_cvb.CamCommonVisionBlox

        • camera_ocv.CamOpenCV

        • camera_irs.CamIntelRealSense

        • radar_ifx.RadarIfxBGT60

      • Sensor related base and wrapper classes

        • sensor_interface.SensorInterface

        • sensor_base.SensorBase

        • sensor_process.SensorProcess

        • sensor_manager.SensorManager

        • camera_base.CameraBase

      • Playback Classes

        • playback.SensorPlayback

        • Deprecated: playback_legacy.SensorPlaybackLegacy

    • ifxdaq.recorder: Recorder modules

    • ifxdaq.tools: Tools & Applications

    • ifxdaq.utils: Utils & helper
